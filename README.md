# Training Planner

Графическое приложение для планирования тренировок. Позволяет добавлять, удалять и фильтровать тренировки по типу и дате. Все данные сохраняются в файл `trainings.json`.

**Автор:** Михаил Колокольников

## Возможности

- Добавление тренировки с указанием даты, типа и длительности.
- Отображение списка тренировок в удобной таблице.
- Фильтрация по типу тренировки (Бег, Плавание, Силовая и т.д.) и конкретной дате.
- Удаление выбранной тренировки.
- Автоматическое сохранение данных в JSON-файл.
- Валидация ввода: дата в формате YYYY-MM-DD, длительность — положительное целое число.

## Требования

- Python 3.6 или выше
- Библиотека `tkinter` (входит в стандартную поставку Python)

## Запуск

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/MPILOT2011/codbudushegotp.git
   cd codbudushegotp
   ```
2. Запустите приложение:
   ```bash
   python main.py
   ```

При первом запуске будет автоматически создан файл `trainings.json` с демонстрационными данными.

## Использование

1. Заполните поля «Дата» (например, `2026-05-12`), «Тип тренировки» (выберите из списка или введите свой) и «Длительность (мин)» (целое число, например, `45`).
2. Нажмите «Добавить» — запись появится в таблице.
3. Для фильтрации укажите нужный тип и/или дату в нижней панели и нажмите «Применить фильтр». Чтобы показать все записи, нажмите «Сбросить».
4. Для удаления тренировки выделите её в таблице и нажмите «Удалить выбранную тренировку».

## Тестирование (примеры)

### Позитивные сценарии
- Добавление корректных данных:
  - Дата: `2026-05-12`, Тип: `Бег`, Длительность: `30` → запись отображается.
  - Дата: `2026-12-31`, Тип: `Плавание`, Длительность: `60` → запись добавлена.
- Фильтрация:
  - Фильтр по типу `Бег` — остаются только записи с типом «Бег».
  - Фильтр по дате `2026-05-12` — остаются записи за этот день.
  - Совместный фильтр: тип `Бег` и дата `2026-05-12`.
- Удаление записи с подтверждением.

### Негативные сценарии
- Пустые поля: появляется предупреждение «Все поля обязательны».
- Некорректный формат даты: `2026/05/12`, `12 мая` — сообщение об ошибке.
- Длительность не положительное целое: `0`, `-5`, `abc`, `4.5` — ошибка валидации.
- Ввод даты в фильтре без фильтрации: проверка и сообщение при неверном формате.

### Граничные случаи
- Длительность `1` (допустимое минимальное значение).
- Очень длинная длительность (например, `999999`).
- Дата 29 февраля високосного года (если `2028-02-29`) — пройдёт проверку, `2027-02-29` — не пройдёт.
- Очистка всех полей фильтра и нажатие «Применить» — покажет все записи.

## Структура проекта

- `main.py` — исходный код приложения.
- `.gitignore` — файл исключений для Git.
- `README.md` — описание проекта.
- `trainings.json` — файл с данными (игнорируется в репозитории).
